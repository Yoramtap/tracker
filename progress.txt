## Codebase Patterns
- For new top-level web pages, follow the route convention `web/src/app/<route>/page.tsx` with a colocated `page.module.css`, and add discoverability via `web/src/app/components/primary-nav.tsx` when it is primary navigation.
- Replay timeline data should be normalized in `web/src/app/replay/data.ts` by combining `getPrdEntries()` and `posts`, then sorted newest-first by parsed date with a deterministic fallback index for ties/unparseable dates.
- Replay timeline presentation and URL filter handling live in `web/src/app/replay/replay-timeline.tsx`; keep `web/src/app/replay/page.tsx` focused on the route shell and server data handoff.
- For static export routes, query-param-driven UI state should use a client component wrapped in `Suspense` (avoid `await searchParams` in the server page component).
- For replay event selection, derive the active event from the filtered list (`selected` fallback to first visible event) instead of syncing selection in an effect; this avoids `react-hooks/set-state-in-effect` lint errors.
