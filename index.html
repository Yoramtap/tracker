Cycle time by team

i

Cycle time per team is computed from lifecycle timestamps.

  • Cycle time = time spent in In Development + UAT, including ongoing work.
  • Shipped counts include only ideas that reached Done through recorded Development/UAT time and not adminned to Done.
  • Aggregated by team using selected sample scope.

i

Workflow breakdown shows average time spent in the tracked board workflow stages for each team.

  • This shows average days spent in each tracked Jira workflow stage.
  • The current stages are Progress and Review, plus QA where that board has a QA column.
  • The data source is board-backed Jira status history over the selected time window.

PR activity by team (Jira proxy)

i

PR inflow, Merged PRs, and Review to merge time (ticket proxy) show deduped Jira-linked pull requests over time by team using Jira-derived proxy dates.

  • Each line is one team.
  • Inflow uses the earliest available PR last update and source-branch last commit. Merged uses Jira PR last update as a merge proxy.
  • Review to merge time uses the first Jira In Review status on the ticket to the linked merged PR proxy date.
  • PRs are deduped before counting, but attribution still depends on the Jira ticket label.
