UAT time by Business Unit

i

UAT time by Business Unit compares average UAT time across Business Units.

  • Use the toggle to compare open in-UAT work vs done work that passed through UAT.
  • Measured in months with a 1-month target.
  • Color shifts from green to yellow to red as UAT time increases.
  • This chart refreshes from current medium+ priority UAT issues plus done items that passed through UAT, grouped by Business Unit.

Workflow breakdown by team

i

Workflow breakdown by team shows average time spent in the tracked Jira workflow stages for each team across the selected rolling window.

  • This shows average days spent in each tracked Jira workflow stage.
  • The current stages are Progress and Review, plus development-side QA checks where that board has a QA column.
  • UAT is tracked elsewhere on the dashboard, so this view stays focused on development-team checks before ship.

Product idea cycle time by team

i

Cycle time per team is computed from lifecycle timestamps.

  • Cycle time = time spent in In Development + UAT, including ongoing work.
  • Shipped counts include only ideas that reached Done through recorded Development/UAT time and not adminned to Done.
  • Use the team toggles for detail, or stay on All teams for the fastest compare view.

How long open ideas stay in each current stage

i

Current lifecycle stage time shows how long open ideas have spent in the stage they are currently sitting in.

  • Use the team toggles to switch between one team and the all-team average.
  • Each stage label includes the current sample count for that stage.
  • This is a current-state view, not a shipped-cycle trend.

Top community contributors

i

Top community contributors ranks assigned contributors by total ticket count.

  • Bars are single-color totals across each contributor's included ticket count.
  • Unassigned and Duplicate-status tickets are excluded from this chart.

Bug backlog trend by team

i

Bug backlog trend tracks open backlog over time by team.

  • Each line is one team.
  • Dashed BC lines show long-standing backlog overlays.

Bugs across teams

i

Current bug backlog summary shows the latest open-bug total, highest/high pressure, and recent backlog change for each tracked team.

  • Percentages are calculated within each team row.
  • Change stacks 30d, 90d, and 6m movement in one place.
  • Use it as the main bug-pressure summary across teams.

PR inflow vs review + QA time

i

PR quality checks maps PR inflow against avg review + QA stage time (days) so it is easier to compare teams at a glance.

  • Each color is one team.
  • Each point shows the team average within the selected lookback window.
  • `PR inflow` reflects linked PR work volume in each sprint bucket.
  • `Avg review + QA stage time (days)` uses the workflow breakdown snapshot for the same window, while PR inflow comes from Jira-linked PR activity buckets.
  • UAT work is tracked separately elsewhere on the dashboard.
