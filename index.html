Backlog Trends dashboard

Shipped last month

i

Shipped last month lets you browse shipped product work by month and year, grouped by mapped team.

  • Use the year switch and month buttons to move through available shipment history.
  • Only months with shipment data are selectable.
  • The detail card lists shipped ideas for the selected month, grouped by team.

Product acceptance time by business unit

i

Product acceptance time by business unit compares average UAT time across Business Units.

  • Use the toggle to compare open in-UAT work vs done work that passed through UAT.
  • Measured in months with a 1-month target.
  • Color shifts from green to yellow to red as UAT time increases.
  • This chart refreshes from current medium+ priority UAT issues plus done items that passed through UAT, grouped by Business Unit.

Development workflow breakdown

i

Development workflow breakdown shows average time spent in the tracked Jira workflow stages for each team across the selected rolling window.

  • This shows average days spent in each tracked Jira workflow stage.
  • The current stages are Progress and Review, plus development-side QA checks where that board has a QA column.
  • UAT is tracked elsewhere on the dashboard, so this view stays focused on development-team checks before ship.

Product delivery and workflow

i

Product delivery and workflow combines shipped delivery timing with the current-state workflow view for the same team set.

  • Use the toggle to switch between Delivery cycle and Ideas workflow.
  • Delivery cycle = time spent in In Development + UAT, including ongoing work.
  • Ideas workflow shows how long open ideas have spent in the stage they are currently sitting in.
  • Use the shared team toggles for detail, or stay on All teams for the compare view.

Top community contributors

i

Top community contributors ranks assigned contributors by total ticket count.

  • Bars are single-color totals across each contributor's included ticket count.
  • Unassigned and Duplicate-status tickets are excluded from this chart.

Bug trends

i

Bug trends combines the sprint trend graph with the current backlog pressure table in one shared panel.

  • Use the toggle to switch between Graph and Table.
  • Graph tracks open backlog over time by team.
  • Table summarizes the latest total, pressure mix, and recent change by team.
  • Dashed BC lines in the graph show long-standing backlog overlays.

Development workflow: PR volume vs Review & QA

i

PR quality checks maps PR inflow against avg review + QA stage time (days) so it is easier to compare teams at a glance.

  • Each color is one team.
  • Each point shows the team average within the selected lookback window.
  • `PR inflow` reflects linked PR work volume in each sprint.
  • `Avg review + QA stage time (days)` uses the workflow breakdown snapshot for the same window, while PR inflow comes from Jira-linked PR activity over time.
  • UAT work is tracked separately elsewhere on the dashboard.
